24 ## OffsetArray
25 """
26 OffsetArray(A, indices...)
27
28 Return an `AbstractArray` that shares element type and size with the first argument but
29 uses the supplied `indices` to infer its axes. If all the indices are `AbstractUnitRange`s then
30 these are directly used as the axis span along each dimension. Refer to the examples below for other
31 permissible types.
32
33 Alternatively it's possible to specify the coordinates of one corner of the array
34 and have the axes be computed automatically from the size of `A`.
35 This constructor makes it convenient to shift to
36 an arbitrary starting index along each axis, for example to a zero-based indexing scheme followed by
37 arrays in languages such as C and Python.
38 See [`Origin`](@ref) and the examples below for this usage.
39
40 # Example: offsets
41
42 There are two types of `indices`: integers and ranges-like types.
43
44 Integers are recognized as offsets, where `0` means no offsets are applied:
45
46 ```jldoctest; setup=:(using OffsetArrays)
47 julia> A = OffsetArray(reshape(1:6, 2, 3), -1, -2)
48 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 0:1, -1:1) with eltype $Int with indices 0:1×-1:1:
49 1 3 5
50 2 4 6
51
52 julia> A[0, 1]
53 5
54 ```
55
56 Examples of range-like types are: `UnitRange` (e.g, `-1:2`), `CartesianIndices`,
57 and `Colon()` (or concisely `:`). A `UnitRange` specifies the axis span along one particular dimension,
58 `CartesianIndices` specify the axis spans along multiple dimensions, and a `Colon` is a placeholder
59 that specifies that the `OffsetArray` shares its axis with its parent along that dimension.
60
61 ```jldoctest; setup=:(using OffsetArrays)
62 julia> OffsetArray(reshape(1:6, 2, 3), 0:1, -1:1)
63 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 0:1, -1:1) with eltype $Int with indices 0:1×-1:1:
64 1 3 5
65 2 4 6
66
67 julia> OffsetArray(reshape(1:6, 2, 3), :, -1:1) # : as a placeholder to indicate that no offset is to be applied to the first dimension
68 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 1:2, -1:1) with eltype $Int with indices 1:2×-1:1:
69 1 3 5
70 2 4 6
71 ```
72
73 Use `CartesianIndices` to specify the coordinates of two diagonally opposite corners:
74
75 ```jldoctest; setup=:(using OffsetArrays)
76 julia> OffsetArray(reshape(1:6, 2, 3), CartesianIndex(0, -1):CartesianIndex(1, 1))
77 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 0:1, -1:1) with eltype $Int with indices 0:1×-1:1:
78 1 3 5
79 2 4 6
80 ```
81
82 Integers and range-like types may not be combined in the same call:
83
84 ```julia
85 julia> OffsetArray(reshape(1:6, 2, 3), 0, -1:1)
86 ERROR: [...]
87 ```
88
89 # Example: origin
90
91 [`OffsetArrays.Origin`](@ref) may be used to specify the origin of the OffsetArray. The term origin here
92 refers to the corner with the lowest values of coordinates, such as the left edge for an `AbstractVector`,
93 the bottom left corner for an `AbstractMatrix` and so on. The coordinates of the origin sets the starting
94 index of the array along each dimension.
95
96 ```jldoctest; setup=:(using OffsetArrays)
97 julia> a = [1 2; 3 4];
98
99 julia> OffsetArray(a, OffsetArrays.Origin(0, 1))
100 2×2 OffsetArray(::$(Array{Int,2}), 0:1, 1:2) with eltype $Int with indices 0:1×1:2:
101 1 2
102 3 4
103
104 julia> OffsetArray(a, OffsetArrays.Origin(0)) # set the origin to zero along each dimension
105 2×2 OffsetArray(::$(Array{Int, 2}), 0:1, 0:1) with eltype $Int with indices 0:1×0:1:
106 1 2
107 3 4
108 ```

136 # checks if the offset may be added to the range without overflowing
137 function overflow_check(r::AbstractUnitRange, offset::Integer)
138 Base.hastypemax(eltype(r)) || return nothing
139 # This gives some performance boost https://github.com/JuliaLang/julia/issues/33273
140 throw_upper_overflow_error(val) = throw(OverflowError("offset should be <= $(typemax(Int) - val) corresponding to the axis $r, received an offset $offset"))
141 throw_lower_overflow_error(val) = throw(OverflowError("offset should be >= $(typemin(Int) - val) corresponding to the axis $r, received an offset $offset"))
142
143 # With ranges in the picture, first(r) might not necessarily be < last(r)
144 # we therefore use the min and max of first(r) and last(r) to check for overflow
145 firstlast_min, firstlast_max = minmax(first(r), last(r))
146
147 if offset > 0 && firstlast_max > typemax(Int) - offset
148 throw_upper_overflow_error(firstlast_max)
149 elseif offset < 0 && firstlast_min < typemin(Int) - offset
150 throw_lower_overflow_error(firstlast_min)
151 end
152 return nothing
153 end

721 else
722 _halfroundInt(v, r::RoundingMode) = div(v, 2, r)
723 end

725 """
726 center(A, [r::RoundingMode=RoundDown])::Dims
727
728 Return the center coordinate of given array `A`. If `size(A, k)` is even,
729 a rounding procedure will be applied with mode `r`.
730
731 !!! compat "OffsetArrays 1.9"
732 This method requires at least OffsetArrays 1.9.
733
734 # Examples
735
736 ```jldoctest; setup=:(using OffsetArrays)
737 julia> A = reshape(collect(1:9), 3, 3)
738 3×3 $(Matrix{Int}):
739 1 4 7
740 2 5 8
741 3 6 9
742
743 julia> c = OffsetArrays.center(A)
744 (2, 2)
745
746 julia> A[c...]
747 5
748
749 julia> Ao = OffsetArray(A, -2, -2); # axes (-1:1, -1:1)
750
751 julia> c = OffsetArrays.center(Ao)
752 (0, 0)
753
754 julia> Ao[c...]
755 5
756 ```
757
758 To shift the center coordinate of the given array to `(0, 0, ...)`, you
759 can use [`centered`](@ref OffsetArrays.centered).
760 """
761 function center(A::AbstractArray, r::RoundingMode=RoundDown)
762 map(axes(A)) do inds
763 _halfroundInt(length(inds)-1, r) + first(inds)
764 end
765 end
766
767 """
768 centered(A, cp=center(A)) -> Ao
769
770 Shift the center coordinate/point `cp` of array `A` to `(0, 0, ..., 0)`. Internally, this is
771 equivalent to `OffsetArray(A, .-cp)`.
772
773 !!! compat "OffsetArrays 1.9"
774 This method requires at least OffsetArrays 1.9.
775
776 # Examples
777
778 ```jldoctest; setup=:(using OffsetArrays)
779 julia> A = reshape(collect(1:9), 3, 3)
780 3×3 $(Matrix{Int}):
781 1 4 7
782 2 5 8
783 3 6 9
784
785 julia> Ao = OffsetArrays.centered(A); # axes (-1:1, -1:1)
786
787 julia> Ao[0, 0]
788 5
789
790 julia> Ao = OffsetArray(A, OffsetArrays.Origin(0)); # axes (0:2, 0:2)
791
792 julia> Aoo = OffsetArrays.centered(Ao); # axes (-1:1, -1:1)
793
794 julia> Aoo[0, 0]
795 5
796 ```
797
798 Users are allowed to pass `cp` to change how "center point" is interpreted, but the meaning of the
799 output array should be reinterpreted as well. For instance, if `cp = map(last, axes(A))` then this
800 function no longer shifts the center point but instead the bottom-right point to `(0, 0, ..., 0)`.
801 A commonly usage of `cp` is to change the rounding behavior when the array is of even size at some
802 dimension:
803
804 ```jldoctest; setup=:(using OffsetArrays)
805 julia> A = reshape(collect(1:4), 2, 2) # Ideally the center should be (1.5, 1.5) but OffsetArrays only support integer offsets
806 2×2 $(Matrix{Int}):
807 1 3
808 2 4
809
810 julia> OffsetArrays.centered(A, OffsetArrays.center(A, RoundUp)) # set (2, 2) as the center point
811 2×2 OffsetArray(::$(Matrix{Int}), -1:0, -1:0) with eltype $(Int) with indices -1:0×-1:0:
812 1 3
813 2 4
814
815 julia> OffsetArrays.centered(A, OffsetArrays.center(A, RoundDown)) # set (1, 1) as the center point
816 2×2 OffsetArray(::$(Matrix{Int}), 0:1, 0:1) with eltype $(Int) with indices 0:1×0:1:
817 1 3
818 2 4
819 ```
820
821 See also [`center`](@ref OffsetArrays.center).
822 """
823 centered(A::AbstractArray, cp::Dims=center(A)) = OffsetArray(A, .-cp)
824
825 centered(A::AbstractArray, i::CartesianIndex) = centered(A, Tuple(i))
